Key Takeaways

  • A vector is a data structure used to store and access elements.
  • A heap sort is a comparison-based sorting algorithm.
  • Runtime (Big O-notation) helps us predict how an algorithm's execution time grows as the input size increases.
  • Not all growth is linear: Selection sort runs in O(n^2), so doubling the input size roughly quadruples the runtime. Heap sort runs in O(n log n), so its runtime grows much more slowly as the input size increases (much faster execution) compared to selection sort. Therefore, doubling the data size doesn't always double the runtime.
