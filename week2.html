Topics Covered

Recursion

There are two core components for recursion to work properly:

  • Base case: the condition that stops recursion
  • Recursive case: the function calls itself, which simplifies the original problem by shrinking the input size.

Divide and Conquer Algorithms

1. Divide: These algorithms break a big problem into smaller subproblems.

2. Conquer: Solve the subproblems (recursively).

3. Merge: Combine the results

Examples are: QuickSort, MergeSort

Quick Sort

Quick sort is a sorting algorithm that:

1. Picks a pivot

2. Partition the array into elements less than the pivot and greater than the pivot

3. Recursively sorts the left subarray

4. Recursively sorts the right subarray

5. Stops when base case is met (size 0 or 1)

Notes:

Quick sort can be slow with a bad pivot (especially on already sorted arrays) but can be strengthened by a randomized pivot (aka Randomized Quick sort).

Runtime:

  • Best case: O(n log n)
  • Average case: O(n log n)
  • Worst case: O(n²)

Merge Sort

Merge sort is a sorting algorithm that:

1. Splits the array in half to create a left and right subarray

2. Recursively sorts the left subarray

3. Recursively sorts the right subarray

4. Merges the two subarrays into one sorted array

5. Stops when base case is met (size 0 or 1)
